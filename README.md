# RE:BOOT - Event Official Site
クラブイベント[RE:BOOT]の公式サイトと、VIP予約を管理するWebシステム。

## 🔗DEMO
- サイト　　https://sho-amano03.github.io/RE-BOOT-vol.2/
- 予約フォーム　　https://forms.gle/ytMwntiaCqJMw6R38
  ※デモ用として公開しています。個人情報の入力はお控えください。

## 概要

ストリートダンスのショーケースだけでなく、プロのDJ、ラッパーなど多様な演者が出演する新感覚のクラブイベントです。
本イベントは1度開催しており、現在は第2回(vol.2)の開催に向けて準備を進めています。
本サイトではイベント情報の掲載に加え、VIP席の予約受付およびメールの自動化機能を実装しています。

## 本サイトの目的

これまでイベント情報はInstagramを中心に発信していましたが、投稿が流れてしまい「必要な情報にたどり着きにくい」という課題がありました。
そこで、情報を整理して確認しやすい公式サイトを作成し、来場者・出演者がスムーズに必要な情報へアクセスできる状態を目指しました。
また、イベント運営の効率化も目的としており、これまでDMで個別対応していたVIP席の予約受け付けをサイト上で一元管理できる仕組みを構築しました。
さらに、予約完了時の自動返信メールや、当選・落選メールの送信機能を実装することで、運営側の作業負担を軽減しています。

## ページ構成、目的

### index.html
- イベント概要
- 開催日時
- 会場情報
- 料金案内
- VIP席案内
- VIP席予約フォーム
- 注意事項

RE:BOOTの公式情報を掲載するトップページです。
イベント概要、VIP席予約、注意事項をまとめて確認できるようにしています。

### css/style.css
サイト全体のデザインを管理するスタイルシートです。
配色や余白、文字サイズ、予約ボタン周りの見た目を整えています。

## 使用技術
- HTML
- CSS
- Google Forms（予約フォーム）
- Google Sheets（データ管理・集計）
- Google Apps Script(メール送信・処理の自動化)

## 技術工夫　/ 学び

### 1.フォーム送信からメール送信までの自動化
Google Formsで受け付けたデータをGoogle Sheetsに反映し、
Google Apps Scriptを用いて受付完了メールを自動返信する仕組みを構築しました。
これにより、手動でのメール対応を削減し、運営の効率化を実現しました。

### 2.データ取得方法の改善による安定化
当初は質問名を基に値を取得していましたが、
質問内容の変更によりデータ取得が不安定になる問題がありました。
そのため、スプレッドシートの列番号を基に値を取得する方法に変更し、データ取得の安定性と保守性を向上させました。

### 3.メール送信の重複防止
送信済みのユーザーに対して重複してメールが送られないよう、送信状態を管理する仕組みを実装しました。

### 4.モバイルファーストを意識したレイアウト調整
スマートフォンでの閲覧を前提とし、画面幅に応じてレイアウトが崩れないよう調整しました。

### 5.検証方法
Chrome DevToolsを使用し、CSSが意図せず上書きされていないかを確認しました。
